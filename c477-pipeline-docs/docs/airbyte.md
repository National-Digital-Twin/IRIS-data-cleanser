# [Why Airbyte?](https://airbyte.com)
<!-- ![Airbyte logo](airbyte.jpg) -->


### <b> [Low code](#low-code)</b>

The reason for Airbyte's emergence as a prominent data integration tool is because historically, consolidating data from various sources for analysis was often a specialized task, requiring significant technical expertise. This situation left many non-technical users dependent on engineers even for basic data operations. Airbyte has been developed to bridge this gap, allowing users without deep technical know-how to connect and manage data sources with greater autonomy.



### <b>[Open Source](#open-source)</b>

Airbyte is better than proprietary software, because it is open-source, fostering an environment where a community of developers and users can contribute to its development. This collaborative approach helps accelerate the enhancement of features and functionalities, often leading to a more innovative and cost-effective solution compared to closed-source alternatives. The trade-off here is between the freedom and cost savings of open-source software and the potentially more structured support that comes with proprietary solutions.


### <b>[Flexibility](#flexibility)</b>

An option to write custom sources in Airbyte is analogous to having an expert chef in a restaurant kitchen. However, not every meal requires gourmet expertise—similarly, not all data integration tasks necessitate custom coding. Airbyte provides a versatile solution by offering ready-to-use connectors for standard use cases, as well as the option for customized connectors to cater to specific needs. Airbyte also integrates with Docker to ensure that custom data connections are stable and consistent. Docker acts as a standardized environment that wraps around Airbyte's connections, akin to a safety net that catches any potential issues that may arise during data transfer, ensuring reliability for the user.


### <b> [Version Control](#version-control)</b>

Moreover, Airbyte employs version control to manage changes in data connections, offering users the ability to track, review, and if necessary, revert to earlier stages of their data pipelines. A robust logging functionality ensures a safety net for users, allowing careful monitoring and swift resolution of issues. However, for those who find detailed logs intimidating or overly technical, Airbyte can customise to the level of detail as defined by the  user.

Airbyte's scheduling capabilities enable automated, timed data operations, much like setting an alarm clock to perform a task without manual intervention. This automation provides regular data updates from various internal systems, external websites, or applications. It is also flexible, granting the ability to manually trigger these processes when needed.

Airbyte's compatibility with workflow orchestration tools such as Apache Airflow and Prefect empowers developers to craft customized data management workflows. This integration allows the seamless insertion of Airbyte into broader, more complex systems where data handling can be personalized to match precise developer and business requirements.
